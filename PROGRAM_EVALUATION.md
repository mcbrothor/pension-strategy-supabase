# 개인 연금 운용 프로그램 평가 및 무료 업그레이드 플랜

## 1. 전제

이 프로그램은 앞으로도 개인이 연금저축/IRP 운용 판단에 참고하는 용도로 사용합니다. 따라서 운용회사급 OMS, 컴플라이언스, 감사 추적, 기관용 보고서 거버넌스, 다중 사용자 권한 체계는 업그레이드 대상에서 제외합니다.

실제 주문은 본 프로그램에서 참고안을 확인한 뒤 MTS에서 직접 실행하는 전제를 유지합니다.

## 2. 개인용 기준 종합 평가

현재 프로그램은 개인 연금 운용 보조 도구로는 방향성이 좋습니다. 단순 자산 입력 앱이 아니라 `일일점검 → 리스크 → 주문계획 → 전략검증 → 리포트 → 설정` 흐름이 있으며, VIX, Fear & Greed, 금리차, 실업률 같은 시장 시그널을 참고해 리밸런싱 판단을 돕습니다.

개인용으로 계속 사용할 경우 핵심 갭은 기관용 주문/컴플라이언스가 아니라 다음 네 가지입니다.

- 보유자산 입력과 성과 관리의 정확도
- 리밸런싱 금액 계산의 실사용성
- 무료 데이터 소스의 안정성 및 출처 표시
- 최소한의 테스트와 운영 안정성

## 3. 범위 재정의

| 영역 | 방향 |
| --- | --- |
| 사용자/계정 구조 | 개인 사용 전제 유지. 다중 사용자, 운용역/승인자 권한 분리는 제외 |
| 포트폴리오 관리 | 무료로 가능한 수준에서 보유자산 정확도, 원가, 현금, 스냅샷 관리 개선 |
| 주문 관리 | 제외. 앱은 참고안만 제공하고 실제 주문은 MTS에서 직접 실행 |
| 리밸런싱 | 무료 개인용 수준에서 계산 정확도와 설명력 개선 |
| 컴플라이언스 | 제외. 단, IRP 70% 같은 사용자 참고 경고는 유지 가능 |
| 리스크 관리 | 무료 개인용 수준에서 실현 변동성, MDD, 상관관계, 집중도 개선 |
| 시장 데이터 | 무료 API/공개 데이터 기반으로 폴백, 캐시, 출처 표시 개선 |
| 성과 측정 | 개선 대상. 원가 기준 손익률, 저장 스냅샷 기준 기간 변화율, 전략 기대수익률 대비 차이 표시 |
| 보고서 | 기관용 보고서 거버넌스는 제외. 현재 PDF에는 성과 측정 요약만 반영 |
| 감사 추적 | 제외 |
| 운영 안정성 | 개인 사용에 필요한 미니멈 수준만 개선 |
| 테스트/검증 | 무료로 가능한 핵심 계산 단위 테스트 중심으로 개선 |

| 리밸런싱 | 목표비중 대비 편차로 매수/매도 금액 산출 | MTS 주문 전에 금액 단위로 참고하기 쉬운 수준 | 최소 주문금액, 현금 여력, 분할 매수 금액 설명이 부족 | 매수/매도 금액을 만원 단위로 정리하고, 분할 매수 시 1회당 금액과 이유를 더 명확히 표시 |

| 시장 데이터 | KIS, Yahoo, CNN, FRED 연동과 일부 캐시 | 무료/공개 데이터 실패 시 폴백하고 사용자가 출처를 알 수 있는 수준 | API 실패 원인과 폴백 상태가 일부만 보임 | 데이터 출처, 갱신 시각, 캐시/실시간 여부를 주요 카드에 표시하고 실패 시 마지막 성공값 사용 여부를 안내 |

| 운영 안정성 | 빌드는 가능하나 런타임 위험 요소 존재 | 혼자 쓰면서 장애 원인을 빠르게 파악하는 수준 | 일부 import/계산 참조 문제, 환경변수 누락 시 안내 부족 | 누락 import와 런타임 위험을 정리하고, API 실패 시 화면 경고를 명확히 유지 |
| 테스트/검증 | 빌드 확인 위주 | 핵심 계산이 바뀌었을 때 오류를 빨리 잡는 수준 | 리밸런싱/성과/리스크 계산 테스트 부족 | 리밸런싱 금액, 수익률, 스냅샷 성과, IRP 위험자산 비율 계산에 작은 단위 테스트 추가 |

## 5. 제외할 항목

다음 항목은 현재 사용 목적상 개발하지 않습니다.

- 운용사급 다중 사용자/역할 분리
- OMS/브로커 주문 연동
- 주문 승인 워크플로우
- 기관용 사전/사후 컴플라이언스 엔진
- 기관용 보고서 승인/잠금/공시 거버넌스
- 감사 추적 시스템
- 수탁사/회계/브로커 데이터 대사
- 기관용 운영 모니터링과 DR 체계

## 6. 우선 업그레이드 순서

### 1단계: 성과 측정 개선

- 원가 기준 평가손익과 평가손익률 표시
- 저장된 스냅샷 기준 기간 변화율 표시
- 전략 기대수익률을 같은 기간으로 환산해 단순 비교
- 현재 PDF 보고서에 위 성과 요약 반영

### 2단계: 포트폴리오 관리 개선

- 자산별 원가 입력 여부를 더 잘 보이게 표시
- 현금MMF와 일반 ETF 입력 방식을 더 명확히 분리
- 저장 전 총자산과 자산군별 비중 확인
- 스냅샷 기준일을 명확히 표시

### 3단계: 리밸런싱 개선

- 매수/매도 금액을 만원 단위로 더 명확히 표시
- 분할 매수 시 1회당 금액과 분할 이유 표시 강화
- 현금 부족 또는 매도 금액보다 매수 금액이 큰 경우 안내
- 목표비중과 현재비중 차이가 작은 경우 "MTS 주문 불필요 가능" 안내

### 4단계: 리스크/시장 데이터 개선

- 리스크 지표의 추정/실현 기반 여부를 화면과 PDF에 표시
- 시세 데이터 기준일과 출처를 더 잘 보이게 표시
- API 실패 시 마지막 성공값/캐시 사용 여부 안내
- 상관계수 계산 실패 시 어떤 종목 데이터가 부족했는지 안내

### 5단계: 테스트/검증 최소화

- 리밸런싱 금액 계산 테스트
- 평가손익률 계산 테스트
- 스냅샷 기간 변화율 계산 테스트
- IRP 위험자산 비율 계산 테스트
- 빌드 확인을 기본 검증 루틴으로 유지

## 7. 결론

이 프로그램은 운용회사 시스템을 따라갈 필요가 없습니다. 개인이 연금 운용 판단에 참고하고, 실제 주문은 MTS에서 직접 실행한다는 전제라면 현재 구조는 적절합니다.

앞으로의 개선 방향은 기관용 기능을 추가하는 것이 아니라, 개인이 무료로 쓸 때 체감되는 정확도와 신뢰도를 높이는 것입니다. 우선순위는 `성과 측정`, `포트폴리오 입력 정확도`, `리밸런싱 참고 금액`, `무료 데이터 출처/폴백 표시`, `핵심 계산 테스트` 순서가 적합합니다.
