# 연금 운용 데스크 사용자 매뉴얼

## 1. 프로그램 개요

이 프로그램은 삼성증권 IRP/연금저축 운용을 가정한 React 기반 연금 포트폴리오 관리 도구입니다.

주요 목적은 다음과 같습니다.

- 현재 연금 포트폴리오의 자산군 비중을 입력하고 저장합니다.
- 선택한 투자 전략의 목표 비중과 현재 비중을 비교합니다.
- VIX, Fear & Greed, 미국 장단기 금리차, 미국 실업률 같은 시장 시그널을 참고해 일일 점검 결과를 보여줍니다.
- 리밸런싱이 필요한 자산군을 매수/매도 체크리스트 형태로 제안합니다.
- 리스크 지표, 백테스트, 월간 보고서, 은퇴 로드맵을 제공합니다.

투자 자문 자동매매 프로그램은 아닙니다. 화면에 표시되는 주문 계획과 리밸런싱 제안은 사용자가 직접 판단하고 증권사 화면에서 별도로 실행해야 합니다.

## 2. 실행 환경

필요한 기본 환경은 다음과 같습니다.

- Node.js 및 npm
- Supabase 프로젝트
- 한국투자증권 Open API 키
- 선택 사항: FRED API 키, Resend API 키

로컬 실행 명령은 다음과 같습니다.

```bash
npm install
npm run dev
```

프로덕션 빌드는 다음 명령으로 확인합니다.

```bash
npm run build
```

검토 시점 기준 `npm run build`는 성공했습니다. 다만 실제 브라우저 렌더링과 서버리스 API 실행은 환경변수와 API 파일 상태의 영향을 받습니다.

## 3. 환경변수

`.env.example` 기준으로 다음 값을 설정합니다.

```bash
VITE_SUPABASE_URL=YOUR_SUPABASE_PROJECT_URL
VITE_SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY

KIS_APP_KEY=YOUR_KIS_APP_KEY
KIS_APP_SECRET=YOUR_KIS_APP_SECRET
KIS_ACCOUNT_TYPE=virtual

FRED_API_KEY=
```

주요 의미는 다음과 같습니다.

- `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`: 로그인, 보유자산, 설정, 스냅샷 저장에 사용합니다.
- `KIS_APP_KEY`, `KIS_APP_SECRET`: 국내/해외 시세, 과거 가격, VIX, 모멘텀 계산에 사용합니다.
- `KIS_ACCOUNT_TYPE`: `virtual`이면 모의투자 API, `real`이면 실전투자 API 주소를 사용합니다.
- `FRED_API_KEY`: 미국 10Y-2Y 금리차와 실업률 조회에 사용합니다.
- `RESEND_API_KEY`: 있으면 주문 완료 알림 메일 발송에 사용하고, 없으면 서버 콘솔에 mock 로그만 남깁니다.

## 4. 데이터 저장 방식

로그인하지 않으면 데모 모드로 동작합니다. 이때 기본 포트폴리오는 사계절 포트폴리오 예시 데이터이며, 실제 계정 저장소에 반영되지 않습니다.

로그인하면 Supabase를 사용합니다.

- `holdings`: 사용자별 보유 종목 저장
- `config`: 사용자별 전략 설정 저장
- `snapshots`: 저장 시점의 총자산과 자산군 비중 기록
- `stock_master`: 종목명, 티커, 자산군 자동완성 데이터
- `decision_logs`: 주문 체크리스트 완료 후 의사결정 로그 저장

`scripts/sync_stocks.mjs`는 네이버 금융 ETF 목록을 가져와 Supabase `stock_master`에 동기화하는 보조 스크립트입니다.

## 5. 기본 사용 흐름

1. 우측 상단의 `로그인` 또는 `설정`으로 이동합니다.
2. `설정 > 계정`에서 로그인 또는 회원가입을 합니다.
3. `설정 > 자산 관리`에서 보유 ETF/현금 자산을 입력합니다.
4. `전략검증`에서 사용할 투자 전략을 선택하고 적용합니다.
5. `일일점검`에서 오늘의 매수/매도/유지 판단을 확인합니다.
6. 필요하면 `리스크`, `주문계획`, `리포트`, `은퇴 로드맵`을 순서대로 확인합니다.
7. 실제 주문은 증권사에서 직접 실행합니다.

## 6. 화면별 사용법

### 6.1 일일점검

앱의 기본 진입 화면입니다.

확인할 수 있는 항목은 다음과 같습니다.

- 오늘의 실행안: 매도, 매수, 유지 건수와 금액
- 실행 우선순위: 자산군별 우선순위, 조정 금액, 분할 매수 횟수
- 포트폴리오 자산 배분: 보유 종목별 현재 비중과 목표 비중
- 시장 상황 진단: VIX 기준 안정/중립/불안/공황 구간
- 평균 상관계수: 보유 종목의 최근 가격 데이터 기반 상관관계
- 시장 센티먼트: CNN Fear & Greed, FRED 수익률 곡선, FRED 미국 실업률
- 복합 시그널 판단: VIX, Fear & Greed, 금리차를 조합한 분할 매수 횟수

버튼은 다음과 같이 사용합니다.

- `VIX 갱신`: VIX 값을 다시 조회합니다.
- `갱신`: 보유 종목 과거 가격을 기반으로 평균 상관계수를 다시 계산합니다.
- `시그널 갱신`: Fear & Greed, 수익률 곡선, 실업률을 다시 조회합니다.
- `리스크 상세`, `주문 계획`, `리포트`: 해당 탭으로 이동합니다.

### 6.2 리스크

포트폴리오의 리스크 지표를 확인하는 화면입니다.

제공 지표는 다음과 같습니다.

- 기대 변동성
- 샤프 지수
- 소르티노 지수
- MDD
- CVaR 95%
- 리스크 기여도
- 최근 MDD 트렌드

`1Y`, `3Y`, `전체` 버튼으로 기간을 선택할 수 있습니다. 과거 가격 데이터가 충분하면 실현 기반으로 계산하고, 데이터가 부족하면 자산군별 추정치 기반으로 표시합니다.

### 6.3 주문계획

리밸런싱 실행용 체크리스트 화면입니다.

확인할 수 있는 항목은 다음과 같습니다.

- 마지막 리밸런싱 날짜와 경과일
- 매도/매수/유지 건수와 금액
- 종합 판단 근거
- 실행 체크리스트
- 유지 종목 목록

체크리스트의 모든 주문 항목을 체크하면 `결정 로그 저장하기 & 메일 알림` 버튼이 나타납니다. 이 버튼은 Supabase `decision_logs`에 실행 내역을 저장하고, `RESEND_API_KEY`가 있으면 이메일 알림을 보냅니다.

현재 코드상 알림 수신자는 `admin@example.com`으로 고정되어 있으므로 실제 운영 전 수정이 필요합니다.

### 6.4 전략검증

투자 전략을 고르고 현재 포트폴리오와 비교하는 화면입니다.

지원 전략은 다음과 같습니다.

- 60/40 포트폴리오
- 영구 포트폴리오
- 영구 포트폴리오 글로벌
- 사계절 포트폴리오
- 올웨더 2유형
- GTAA 5
- 듀얼 모멘텀
- LAA
- DAA

전략은 크게 두 종류입니다.

- 고정비중 전략: 정해진 목표 비중을 바로 적용합니다.
- 모멘텀 전략: 시장 데이터를 조회해 동적 비중을 계산합니다.

`IRP`, `연금저축`, `혼합` 버튼으로 계좌 유형을 바꾸면 IRP 위험자산 한도 적합 여부를 확인할 수 있습니다. IRP는 위험자산 70% 한도를 기준으로 경고합니다.

`전략 검증 실행` 버튼은 보유 종목 과거 가격을 조회해 백테스트와 Walk-Forward 테스트를 실행합니다.

### 6.5 리포트

월간 보고서 형태로 포트폴리오 요약을 보여줍니다.

포함 내용은 다음과 같습니다.

- 총 자산
- 기대 수익률
- 기대 변동성
- 샤프 지수
- 매니저 인사이트
- 전략 및 리스크 현황
- 자산별 현재 비중, 목표 비중, 편차, 평가금액, 수익률
- 은퇴 준비도 진단

`보고서 출력 (PDF)` 버튼은 새 창을 열고 브라우저 인쇄 기능을 호출합니다. PDF 저장은 브라우저의 인쇄 대화상자에서 선택합니다.

### 6.6 설정

설정 화면은 네 개의 서브탭으로 구성됩니다.

`자산 관리`

- 보유 ETF 또는 현금MMF를 추가합니다.
- 종목명 자동완성 또는 티커 입력을 사용할 수 있습니다.
- `시세 자동조회`는 `/api/kis-price`를 통해 KIS를 먼저 시도하고 실패하면 Yahoo Finance를 사용합니다.
- 현금MMF는 티커 없이 금액만 입력합니다.
- 보유 자산은 수정, 삭제, 전체 삭제할 수 있습니다.

`알림 설정`

- 손절 기준을 설정합니다.
- MDD 제한선을 설정합니다.
- 현재 경고 목록을 확인합니다.

`은퇴 로드맵`

- 현재 나이, 은퇴 나이, 기대 수명
- 현재 잔고, 월 납입액, 목표 생활비
- 물가상승률
- 투자 전략
- 명목 금액/실질 구매력 기준

위 값을 조정해 필요한 은퇴 자산과 달성 가능성을 시뮬레이션합니다.

`계정`

- Supabase Auth 기반 로그인, 회원가입, 로그아웃을 처리합니다.

## 7. 시장 데이터와 API

프로그램은 다음 API를 사용합니다.

- `/api/vix`: KIS에서 VIX 조회 후 실패하면 Yahoo Finance로 폴백합니다. 15분 캐시를 사용합니다.
- `/api/kis-price`: 국내/해외 종목 현재가 조회. KIS 실패 시 Yahoo Finance로 폴백합니다.
- `/api/kis-history`: 최근 60영업일 가격 데이터 조회. 리스크, 상관계수, 백테스트에 사용합니다.
- `/api/market-signals`: CNN Fear & Greed, FRED 10Y-2Y 스프레드, FRED 실업률을 병렬 조회합니다.
- `/api/momentum`: GTAA 5, 듀얼 모멘텀, LAA, DAA의 동적 목표 비중을 계산합니다.
- `/api/notify`: 주문 완료 로그 저장 후 이메일 알림에 사용합니다.

외부 API가 실패하면 화면에 오류 또는 폴백 상태가 표시될 수 있습니다. 모멘텀 전략은 API 실패 시 직전 유효 타깃 또는 전략 기본 비중으로 대체될 수 있습니다.

## 8. 리밸런싱 판단 기준

주문 제안은 다음 조건을 종합해 생성됩니다.

- 목표 비중 대비 현재 비중 이탈
- 마지막 리밸런싱 이후 경과일
- VIX 시장 구간
- Fear & Greed 점수
- 10Y-2Y 수익률 곡선
- 현재 MDD와 MDD 제한선
- 자산별 손절 기준
- IRP 위험자산 70% 한도
- 평균 상관계수

매수 대상은 시장 불확실성이 높을수록 더 많은 횟수로 분할 매수하도록 제안합니다.

## 9. 운영 전 주의사항

다음 항목은 실제 사용 전 확인이 필요합니다.

- `api/vix.js`가 `./utils/kis-auth.js`를 import하지만 현재 `api/utils/kis-auth.js` 파일은 없습니다. 배포 환경에서 서버리스 함수 로딩 오류가 날 수 있습니다.
- `src/App.jsx`에서 `annualExpRet`가 정의되기 전에 참조됩니다. 빌드는 통과했지만 브라우저 런타임 오류 가능성이 있으므로 보정이 필요합니다.
- Supabase 테이블 스키마와 RLS 정책이 코드의 쿼리와 맞아야 합니다.
- `decision_logs` 저장에는 현재 `user_id`가 포함되지 않습니다. 다중 사용자 운영 시 로그 소유권 설계가 필요합니다.
- 주문 알림 수신자가 `admin@example.com`으로 고정되어 있습니다.
- `scripts/sync_stocks.mjs`의 자산군 자동 분류는 단순 키워드 기반입니다.
- 일부 UI 스타일 속성에 `justifyBetween`처럼 CSS 속성명이 잘못된 부분이 있습니다. 치명적 기능 오류는 아닐 수 있지만 레이아웃 의도와 다르게 보일 수 있습니다.
- 금융 데이터와 리스크 지표는 추정치 또는 외부 API 응답에 의존하므로 실제 투자 판단의 유일한 근거로 사용하면 안 됩니다.

## 10. 문제 해결

로그인이 안 될 때:

- `.env.local`의 Supabase URL과 anon key를 확인합니다.
- Supabase Auth에서 이메일 로그인 설정을 확인합니다.

시세 조회가 안 될 때:

- KIS API 키와 `KIS_ACCOUNT_TYPE`을 확인합니다.
- 국내 종목은 6자리 종목코드, 해외 종목은 티커를 입력합니다.
- KIS 실패 시 Yahoo 폴백도 실패하면 네트워크 또는 종목 코드 문제일 수 있습니다.

시장 센티먼트가 일부만 나올 때:

- Fear & Greed는 키 없이 조회하지만, 수익률 곡선과 실업률은 `FRED_API_KEY`가 필요합니다.
- 한 데이터 소스가 실패해도 나머지는 표시될 수 있습니다.

모멘텀 전략 비중이 이상할 때:

- `/api/momentum` 호출 상태를 확인합니다.
- KIS, Yahoo, FRED 데이터가 일부 실패하면 폴백 로직이 적용될 수 있습니다.
- 화면에 `폴백 모드`가 표시되면 직전 유효 타깃 또는 기본 비중을 사용 중입니다.

보고서 출력이 안 될 때:

- 브라우저 팝업 차단을 해제합니다.
- 새 창이 열린 뒤 인쇄 대화상자에서 PDF 저장을 선택합니다.
